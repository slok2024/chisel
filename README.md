

这个项目是一个基于 Chisel 内核定制的 可视化内网穿透与运维工具。

它将强大的命令行隧道工具 Chisel 封装进了一个现代化的 GUI 界面中，旨在解决运维人员在复杂网络环境下进行流量转发、内网穿透和加密通信时，频繁输入繁琐命令、难以管理进程以及架构识别错误等痛点。

以下是该项目的核心组成部分和特点：

1. 核心底座：Chisel (Go 语言编写)
Chisel 是一个通过 HTTP 进行传输，并使用 SSH 进行安全保护的高性能 TCP/UDP 隧道工具。

隐蔽性强：流量经过 HTTP 伪装，可以穿透大多数只允许 Web 流量的防火墙。

安全性高：所有通信都经过 SSH 加密，支持指纹验证和用户认证。

功能全：支持正向端口转发、反向隧道（Reverse Tunnel）以及动态 SOCKS5 代理。

2. 智能管理端 (Python GUI)
这是你目前正在使用的可视化界面，其核心功能包括：

架构自适应逻辑：针对你之前的特殊需求，程序会通过 Windows 系统环境变量（而非 Python 库）精准探测真机是 64 位还是 32 位系统，从而自动调用对应的 chisel_x64.exe 或 chisel_x86.exe，避免了在 64 位系统上误用 32 位环境导致的兼容性问题。

参数逻辑校验：根据 参数.txt 的文档规范，UI 严格限制了参数的输入顺序（Options -> URL -> Remotes），解决了 Chisel 命令行解析报错的问题。

进程“强力”控制：集成了 Windows 原生的 taskkill 指令，一键强制终结进程树，彻底关闭所有挂起的隧道和占用的端口。

实时监控：提供了一个绿色字符风格的控制台日志窗口，实时回显内核运行状态和连接信息。

3. 应用场景
内网穿透：将内网中的 RDP（远程桌面）、SSH 或数据库端口映射到具有公网 IP 的服务器上。

隐蔽维护：利用 --backend 功能将隧道隐藏在正常的网页服务之后，躲避安全监测。

运维跳板：在跳板机上开启 server --reverse，通过一个连接点管理整个内网节点的访问。

4. 项目结构
主程序：chisel.py（或打包后的 ChiselManager.exe）。

内核文件：chisel_x64.exe 和 chisel_x86.exe（需放在同级目录）。

兼容性：支持 Windows 7 到 Windows 11 全系列系统（Win7 需 Python 3.8 环境）。

总结来说，这是一个专为运维打造的“瑞士军刀”级网络工具，既保留了 Go 内核的高效，又提供了 Python GUI 的便捷。
